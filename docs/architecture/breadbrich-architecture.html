4. Dreaming Cycle

Observer

Model: Haiku

Cadence: Every 15 min, per chat

Reads new messages since last run. Extracts facts: decisions, commitments, unresolved threads. Appends to groups/{chat}/observations.md (dated, severity-marked, pointer to source).

Always-append → stable cache prefix → 4-10× token savings on top of raw compression.

Reflector

Model: Sonnet

Cadence: Daily 2am

Compresses observations older than 7 days. Detects duplicates (tasks with similar titles, same person under multiple names). Rebuilds MEMORY.md pointer index per chat. Flags detected dupes to reflector-queue.md for human review.

Curator

Model: (no LLM)

Cadence: Weekly Sun 3am

Tiers KB: hot ≤7d warm ≤30d cold >30d (archive). Purges promoted observations. DB integrity check + backup rotation.

Never deletes — only archives.

6. Migration Phases

PHASE 0 · IN PROGRESS
Consolidation + safety net
Docs, backup, drain, staging. All read-only on prod. Low risk
PHASE 1
Cosmetic rename
systemd Alias=breadbrich.service for breadbrich.service. Cloudflare tunnel label. Trivially reversible. Very low risk
PHASE 2
the central orchestrator orchestrator
SDK init, routing engine, Observer/Reflector. Thin-forwarder containers. Medium risk · ~5min rollback
PHASE 3
Filesystem rename
/opt/breadbrich → /opt/breadbrich via copy + symlink bridge. 1-month rollback window. Medium risk
PHASE 4 · OPTIONAL
User account rename
breadbrich → breadbrich user. High risk, low reward. Recommended: skip. System usernames are invisible to end-users.
PHASE 5
Cleanup
Drop aliases, delete /opt/breadbrich.old, tag v3.0.0. Each step irreversible — only after verify window. Irreversible
